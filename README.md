# FE-A
채용 과제 | 라이브클래스

<h2>프로젝트 개요</h2>
<p>해당 내용은 온라인 강의 신청 폼에 관한 것이며 신청은 개인과 그룹으로 나뉜다. 그룹의 경우 최대 10명까지 등록 가능하며 점진적인 스탭을 밟아가며 최종에 이른다.</p>

<h2>기술 스택</h2>
<p>React, Vite, typsscript, css, html</p>

<h2>실행방법</h2>
<p>해당 git url clone 이후 npm install 후 npm run dev를 통해 진행된다.</p>

<h2>프로젝트 구조 설명</h2>
<ol>
  <li>초기 화면은 카테고리 별로 강의들을 그리드 형태로 보여주며 20개씩 묶어서 페이지 형식의 구성을 띈다. 기본값은 모든 카테고리의 강의들이 노출되며 상단 카테고리를 통해 분류한다.</li>
  <li>사용자가 강의 카드에 마우스를 올릴 경우 그에 대한 핵심 정보를 포함한 popup이 뜨고 개인과 그룹의 토글 방식의 선택을 통해 수강신청 버튼을 유도한다.</li>
  <li>개인과 그룹의 지원서 페이지를 따로 관리하며 그룹은 개인의 신청 내역을 포함한다.</li>
  <li>그룹 신청의 경우 최소 2명에서부터 최대 10명까지 추가 버튼을 통해 등록 가능하며 삭제기능을 가지고 있다. 하나라도 신청폼의 조건을 만족하지 않을 경우 다음으로 넘어가지 않고 오류 메시지를 띄운다.</li>
  <li>신청서가 통과될 경우 최종적으로 요약 정보를 제공하며 약관동의 절차를 진행한다.</li>
  <li>수강신청이 완료된 후 최종 페이지에 도착하며, 신청번호와 완료된 시점 등을 표시한다.</li>
</ol>

<h2>요구사항 해석 및 가정</h2>
<p>참고 api에서의 startDate와 endDate가 수강신청 등록기간으로 받아들였고 지원서 폼 하단 왼쪽에 마감기한을 별도로 표시하였다.</p>
<p>다단계의 방식을 화면을 넘기는 것으로 받아들여 페이지 형식을 구성했다.</p>
